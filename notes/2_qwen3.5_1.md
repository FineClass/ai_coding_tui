### 2. TUI 技术选型建议

```
推荐方案：Python + Textual + Rich
├── Textual：现代 TUI 应用框架（支持鼠标、异步、组件化）
├── Rich：富文本渲染（表格、进度条、语法高亮、traceback）
└── 优势：纯 Python、跨平台、可同时在终端和浏览器运行
```

**备选方案**：
- **Rust + Ratatui**：性能更高，适合系统级工具
- **Go + Bubble Tea**：函数式 TUI 框架，简洁高效

### 3. 轻量级隔离环境设计

你提到的 `switchroot` 思路很巧妙，比 Docker/VM 更轻量。建议采用**分层隔离策略**：

```
┌─────────────────────────────────────────────────────────┐
│                    AI Coding System                      │
├─────────────────────────────────────────────────────────┤
│  TUI Layer (Textual)  │  会话管理 │  命令历史 │  状态显示  │
├─────────────────────────────────────────────────────────┤
│                    Agent Core                            │
│  (任务规划 │ 工具调用 │ 代码生成 │ 执行验证)              │
├─────────────────────────────────────────────────────────┤
│                 隔离执行层 (Isolation)                   │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐  │
│  │  namespace  │  │  chroot     │  │  资源限制        │  │
│  │  (PID/Net)  │  │  (文件系统)  │  │  (cgroups)      │  │
│  └─────────────┘  └─────────────┘  └─────────────────┘  │
├─────────────────────────────────────────────────────────┤
│                    工具层 (Tools)                        │
│  文件操作 │ 命令执行 │ Git │ 包管理 │ 测试运行 │ 网络请求   │
└─────────────────────────────────────────────────────────┘
```

## ⚠️ 关键注意事项
