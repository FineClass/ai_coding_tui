# 📋 Claude Code 开源实现调研报告

## 一、核心发现

### 1. Claude Code 本身定位

| 属性 | 说明 |
|------|------|
| **出品方** | Anthropic (2025年推出) |
| **开源状态** | 客户端开源，模型云端运行 |
| **交互方式** | 纯 CLI 终端 |
| **核心能力** | 读/改代码、跑命令、Git 操作、多文件上下文 |

> ⚠️ **注意**：Claude Code 不是完全开源项目，其核心模型运行在 Anthropic 云端。但有多个开源替代方案和扩展工具。

## 三、Open Interpreter 深度分析（最接近的开源实现）

### 架构特点
```
┌─────────────────────────────────────────────────────────┐
│                   Open Interpreter                       │
├─────────────────────────────────────────────────────────┤
│  自然语言输入  →  LLM  →  代码生成  →  本地执行  →  结果反馈  │
├─────────────────────────────────────────────────────────┤
│  支持语言：Python / JavaScript / Shell / 等              │
│  支持模型：GPT-4 / Claude / 本地模型 (Ollama/vLLM)        │
│  执行环境：用户本机（可配置沙箱）                        │
└─────────────────────────────────────────────────────────┘
```

### 核心特性
- ✅ **本地执行**：代码在用户机器上运行，非云端沙箱
- ✅ **多模型支持**：可接 OpenAI、Claude、本地 Qwen/Ollama
- ✅ **计算机控制**：可截图、OCR、鼠标键盘操作（Computer API）
- ✅ **配置灵活**：系统提示、权限白名单、资源限制可定制

### 项目结构参考
```
open-interpreter/
├── interpreter/          # 核心解释器
├── terminal_interface/   # 终端交互层
├── computer/             # 计算机控制 API
├── profiles/             # 配置文件模板
└── tools/                # 工具集合
```

## 四、Aider 深度分析（终端结对编程）

### 核心优势
- 🎯 **Git 友好**：所有修改自动 commit，可追溯
- 🎯 **多模型路由**：支持 Claude/GPT/本地模型切换
- 🎯 **项目理解**：自动索引代码库，上下文管理优秀
- 🎯 **纯 Python**：易于理解和扩展

### 命令示例
```bash
aider                       # 进入交互模式
aider --model claude-3-5    # 指定模型
aider --git                 # 启用 Git 集成
aider file1.py file2.js     # 指定关注文件
```

## 五、MCP (Model Context Protocol) 协议

Anthropic 推出的**上下文交互标准协议**，是 Claude Code 生态的关键：

| 组件 | 作用 |
|------|------|
| **MCP Server** | 提供工具/资源/提示词服务 |
| **MCP Client** | 连接服务器，暴露给 LLM 使用 |
| **配置文件** | `~/.claude.json` (用户级) / `.mcp.json` (项目级) |

### 对你的项目启示
```yaml
# 可参考 MCP 设计你的工具协议
tools:
  - file_read
  - file_write
  - shell_exec
  - git_operation
  - test_run
  
resources:
  - project_files
  - command_history
  - session_state
```

## 九、关键设计决策

| 决策点 | 建议 | 理由 |
|--------|------|------|
| **语言选择** | Python | 生态丰富、Textual 支持好、快速原型 |
| **隔离方案** | namespace+chroot | 比 Docker 轻量，比无隔离安全 |
| **模型接入** | 多后端抽象 | 避免锁定，支持本地/云端切换 |
| **交互方式** | TUI 优先 | 符合 CLI 定位，比纯文本更友好 |
| **配置格式** | YAML + JSON | 人类可读 + 机器友好 |
