    想法：
      GUI是一个使用者与模型之间的交互环境的可视化，交互环境是一个针对当前项目的“虚拟隔离”环境，能提供给模型各种已有工具进行使用。
      虚拟隔离环境 类似docker/VM，但那太重太复杂，这里类似switchroot切换根文件系统的方式。

### 4.
    针对代码项目，目前我可以将其分为两类：
    1.“新功能”或“优化功能”的流程
    (1)需求探讨、确定方案
    产物：详细的需求文档
    (2)设计
    产物：详细的设计文档
    (3)任务迭代
    a.根据设计文档划分任务
    b.任务分配
    c.执行与反馈的局部迭代
    这是当前任务部分的局部迭代。
    编码、测试、反馈，迭代至该任务结果满足要求。
    现在很多模型都在这个环节上执行差。
    d.整体的测试与反馈的迭代
    现在很多模型都在这个环节上仅仅是修复问题，没有完全按照设计文档的要求进行测试，或测试结果不符合设计文档要求。
    这里是完整程序的全局迭代。
    2.“修复缺陷”的流程
    (1) 明确问题，确定问题点
    (2) 展示修复方案
    (3) 执行 编码与测试的迭代
