# ACPRE(AI Coding Project Runtime Environment) TUI

## 项目现状

ACPRE TUI 是一个基于 C++ / FTXUI 的终端工作流界面，用于展示“新功能 / 缺陷修复”两类软件开发流程。

当前版本已经从“模拟 AI 阶段输出”调整为“基于当前目录真实工程信息的阶段执行”。

## 最新更新（2026.03.26）

### 已实现

- 读取当前工作目录，并在界面中显示工作目录路径
- 扫描当前目录下的文件与子目录，生成阶段摘要
- 统计文件扩展名分布，用于辅助展示工程结构
- 在阶段执行时调用真实构建命令：
  - 分析阶段 / 方案阶段读取 `make -n` 预览结果
  - 执行阶段优先尝试调用 `claude -p` 触发真实代码修改任务
  - 若未检测到 `claude` 命令，则自动回退为本地 `make` 构建检查
  - 执行结果会展示执行器是否可用、执行摘要、关键信号与构建结果
- 验收改为人工确认模式，需逐项确认后才能完成阶段
- 复核阶段的状态语义已统一为“等待验收确认 / 建议复核 / 存在失败信号”

### 已移除的模拟行为

- 模拟 AI 对话日志
- 固定伪造的需求 / 设计 / 实现输出
- 自动将所有验收项标记为已确认
- 与当前目录无关的示例任务树和示例 Bug 信息

### 当前交互方式

- `Space`：启动或推进当前阶段
- `T`：在复核阶段确认下一项验收
- `C`：在可完成时完成当前阶段
- `R`：重试当前阶段
- `I`：查看迭代历史
- `V`：查看全局复核
- `Ctrl+Q`：退出

### 当前限制

当前版本已经具备真实目录上下文感知、真实构建结果展示，以及执行阶段触发 Claude Code 的能力，但仍不是一个自动修改任意工程代码的通用代理。

目前主要能力：
- 真实目录上下文感知
- 执行阶段调用 `claude -p` 触发真实代码修改任务
- `claude` 不可用时自动回退到本地构建检查
- 基于验收项人工确认与复核状态推进阶段

后续可继续扩展：
- 在 TUI 中增加执行器选择与自定义命令配置
- 在 TUI 中增加文件选择、目标编辑、测试入口与结果记录

## 使用方式

```bash
make
./acpre_tui
```
